Admin API Documentation

Base URL

https://shoppify-seven.vercel.app/admin

Districts

Add District

POST /district

Request Body:

{
  "name": "District Name"
}

Response: 201 Created

{
  "_id": "district_id",
  "name": "District Name",
  "createdAt": "timestamp",
  "updatedAt": "timestamp"
}

Get All Districts

GET /district

Response: 200 OK

[
  {
    "_id": "district_id",
    "name": "District Name",
    "createdAt": "timestamp",
    "updatedAt": "timestamp"
  },
  ...
]

Update District

PUT /district/:id

Request Body:

{
  "name": "Updated District Name"
}

Response: 200 OK

{
  "_id": "district_id",
  "name": "Updated District Name",
  "createdAt": "timestamp",
  "updatedAt": "timestamp"
}

Delete District

DELETE /district/:id

Response: 200 OK

{
  "message": "District deleted successfully"
}

Error Responses

In case of errors, the API will return a JSON object with a "message" field describing the error: 

{
  "message": "Error description"
}

Common HTTP status codes:

  • 400 Bad Request: Invalid input data
  • 404 Not Found: Resource not found
  • 500 Internal Server Error: Unexpected server error

Authentication

This API currently does not require authentication. Ensure to implement proper authentication and authorization before deploying to production.
